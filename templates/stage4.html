Introduction to Networks

Intro to Networks part-1

A Network is a group of entities that can communicate, even though they are not all directly connected. At least 3 entities that can communicate with each other without a direct connections to each others.

There are somethings we need to make a network.

      Ways to encode and interpret messages
      Ways to route messages
      Rules for deciding who gets to use resources

There are two main ways to measure a newtwork. One way is using Latency and the other way is to use Bandwidth. Latency is the time that is takes a message to get from the source to the destination. Which is a measure of time now most commonly measured in milliseconds. Bandwidth is the amount of information that can be transmitted per unit time. Usually measured in bits per second more commonly written as Mbps.

A Bit is the smallest unit of information. One bit lets us decide between two choices hence 0 and 1 for "yes" and "no". Measuring a bandwidth we need to know how many bits we can send per second.

A Protocol is a set of rules that people agree to, that tell you how two entities can talk to each other. For the web the protocol gives rules on how the client and the server communicate with each other. Client is the webbrowser that we are viewing and the server is the website we are at. The protocol says that we have to send a message in a particualr way. The protocol that we use on the web is HTTP("Hypertext Transfer Protocol") this sends the Get message to the client and then the client sends back what we wanted.

Make the Internet Work for You

A URL stands for Uniform Resource Locator. An example URL: http://www.udacity.com/ . This consists of three parts. The Protocol wich is "http" the host which is "www.udacity.com" which is the domain name which is the description of the location of the page or file we want and the path "/" which allow

Query parameters (GET parameters), an example:

https://example.com/foo?P=1

This ?P=1 adds a Parameter whos name is P and value is 1. The format is "Name = value" we could add a new parameter by using ampersands "&". Parameters are added information the server gets like cache. A Cache is something that stores data so that you don't have to rectrieve it later. It can be used to make data requests faster.

Fragments are separated from the URL with a "#" and is generally used to reference a particular part of the page you are looking at. It is not sent to the server when you make a request, it purely exists in the browser. Fragments follow the query parameters, they come last.

URL have ports. When you make a request to a server, http://localhost:8000/ this is the host, the name of the machine or the location of the machine you are requesting. You need two things to make an internet connection which are the address of the machine with is represented by the host then the port with is ":8000" it is 80 by default but we can use a different port.

HTTP (HyperText Transfer Protocol). The request from our browser for any URl begins with a request line, it will look something like this:

GET /foo HTTP/1.1
there are three main parts for the request line. Method which in this case is "GET" which is the type of request we are making to the server. The Path is "/foo" which is the actual document we are requesting from the server. And the version which is "HTTP/__" 1.1 or 1.0.

When we make a request it is followed by Headers: Name: value.For example:

Host: www.example.com
User-Agent: chrome.

HTTP Responses: When we make a request to a server we also get a response. A basic HTTP response looks similar to the request. So the response to the request line is HTTP/1.1 200 ok it will always be the same version as the request and then we get the status code which in this case is 200 and then the reason phrase which is in english ok. Here are some of the most common status codes.

      200 ok
      302 Found
      404 Not found
      500 Server Error
The staus line for HTTP Responses, like the request have headers.

A purpose of a Server is to respond fo HTTP requests. There are two main classification of types of responses the server will do which are Static and Dynamic. Static are pre-written file that the server just returns like an image. Dynamic are requests where the response is built on the fly by a program thats running. These programs are called Web Applications which is by definition a program that generates content.

We can make a radio button to where only one button can be selected by setting the name attribute on all the options to be the same. But this will not tell us which button was selected, this only tells us that a button was selected. To know which one was selected we will add another attribute to the input tag called value with this we can set the names of the buttons to be the same but the value be different so we know exactly which button was selected.

So far we have three options to select but in the browser we see three buttons but the user does not know what the values are. We can change this by using the Label element so we can inform the user what they are selecting.

We can create a dropdown box by using the select element then in that element we have option elements.

There is a attribute called action and this controls where a form submits to.

    GET
      Parameters are in URL

      Generally ok to cache

    POST

Validation

Validation means verifying on the server side that what we receive is what we expected to receive. So for instance we can ask the user to input a number from 1 to 10, we can validate that the user inputs a number in that range by writing code to check the input make sure it is in fact a digit between 1 and 10 and if it is not, we output some kind of error message. This does not allow the user to give us bad content, so whatever they enter will not affect our code it will just give the user feedback that what they entered it invalid.

String substitution is a neat feature that allows the user to write %s then %variable and this will sub in what ever the variable is wheather it be a string or text, and it will take the place of %s. We can also use this to substitute multiple variables in this format: "text %s text %s" %(var1, var2) this alows us to sub in two variables at a time, there can be more than just two variables.

When we get inputs from the user inputing text through the browser when he enters the wrong value it reloads the page but doesn't save the info he already entered in that is correct. We can change that by using the input value <input value="%s"> this will save the information that was entered previously with a defalt value.

Escaping is a really cool feature it allows us to make the input data safer by not letting whatever get inputed to mess with our HTML code. We can represent some the symbols that are used to write code with a safe piece of code specific to each symbol to where it will render the symbol in the browser but not in our code.

Databases

Databases are how we store all the information we are getting from the users. Ther are different types of databases:
relational databases(SQL) which include MySQL, Oracle, and SQLite. These all use tables.

Tables tables have columns and rows to help us track different things like to see how many votes user get from submitting links.

Downsides of querying data by hand is that it is error prone, it is a very tedious process, and is very slow when processing a lot of requests.

SQL (Structured Query Language) its a language for expressing queries. We ask questions to the database to get information. It was invented in the 1970s. We can write in this format: SELECT * From links WHERE id=5;

Joins is a SQL statement where we scan two different tables to link them together.

Indexes increases the speed of queries becauase instead of just scaning the query to find the one that matches what we need, we use indexes to jump straight to where we desire. Hashtables are not sorted but the look ups are faster, Trees are sorted but lookups are slower.

ACID stands for:

  • Atomicity
    • - all parts of a transaction succeed or fail together
  • Consistency
    • - the database will always be consistent
  • Isolation
    • - no transaction can interfere with another's
  • Durability
    • - once the transaction is committed, it won't be lost.
